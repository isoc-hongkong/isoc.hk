# Test 03 — Live WP Sitemap vs Astro URL Coverage

**Effort:** ~10 min | **Prereqs:** Live WP site reachable + `dist/` built | **Tools:** WebFetch, Bash

## Goal

Every URL published on the live WordPress site must be reachable on the Astro site — either as a direct page (200 OK) or via a redirect in `netlify.toml`. URLs that fall through the cracks become 404s that hurt SEO and break bookmarks/backlinks.

## Context

- Live WP site: `https://www.isoc.hk`
- Astro site URL: `https://www.isoc.hk` (same domain, will be deployed to replace WP)
- Astro sitemap: `dist/sitemap-0.xml` (generated by `@astrojs/sitemap`)
- Redirects: `netlify.toml` (auto-generated from a script, covers `?p=N` shortlinks + WP patterns)
- The live WP sitemap is at `https://www.isoc.hk/sitemap.xml` (check — may be a sitemap index pointing to `/sitemap-0.xml` or `/post-sitemap.xml`)

## Steps

### Step 1 — Fetch the live WP sitemap

Using WebFetch, fetch: `https://www.isoc.hk/sitemap.xml`

If this is a sitemap index (contains `<sitemapindex>` with `<loc>` entries), fetch each child sitemap too. Common WP sitemap plugins produce:
- `/sitemap.xml` → index
- `/post-sitemap.xml` → posts
- `/page-sitemap.xml` → pages

Collect all `<loc>` URLs into a list. Remove the `https://www.isoc.hk` prefix to get just the paths.

Save to `/tmp/wp_urls.txt`:
```
/about/
/news/board-of-directors-2024-25/
...
```

### Step 2 — Build the Astro reachable-URL set

**Direct pages from dist:**
```bash
find /Users/chpapa/Workspaces/isoc.hk/astro-site/dist -name "index.html" \
  | sed 's|/Users/chpapa/Workspaces/isoc.hk/astro-site/dist||;s|/index.html|/|' \
  | sort -u > /tmp/astro_direct.txt
echo "/" >> /tmp/astro_direct.txt
```

**Redirect destinations from netlify.toml (all non-query-param rules):**
```bash
# Extract `from` paths (patterns like /author/*, /20:year/ are prefix patterns)
grep -A2 '^\[\[redirects\]\]' /Users/chpapa/Workspaces/isoc.hk/astro-site/netlify.toml \
  | grep 'from' | grep -v 'query' \
  | sed 's/.*= "//;s/".*//' | sort -u > /tmp/netlify_froms.txt

# Also extract `to` destinations for ?p=N rules (the actual target URLs)
grep -A3 '^\[\[redirects\]\]' /Users/chpapa/Workspaces/isoc.hk/astro-site/netlify.toml \
  | grep '"to"' | sed 's/.*= "//;s/".*//' | sort -u > /tmp/netlify_tos.txt

cat /tmp/astro_direct.txt /tmp/netlify_tos.txt | sort -u > /tmp/astro_reachable.txt
wc -l /tmp/astro_reachable.txt
```

### Step 3 — Diff: WP URLs not covered by Astro

```bash
# WP paths that are NOT in the Astro reachable set
# (Note: WP paths with ?p=N would be in wp_urls.txt as /?p=N form — check separately)
comm -23 \
  <(sort /tmp/wp_urls.txt) \
  <(sort /tmp/astro_reachable.txt) \
  > /tmp/missing_urls.txt

wc -l /tmp/missing_urls.txt
cat /tmp/missing_urls.txt
```

### Step 4 — Verify `?p=N` shortlink coverage

The WP sitemap typically does NOT include `/?p=N` URLs (those are redirects, not canonical). But search engines and old backlinks may use them. Verify the `netlify.toml` covers all the post IDs in the content collection:

```bash
# Extract all wpId values from news frontmatter
grep -rh "^wpId:" /Users/chpapa/Workspaces/isoc.hk/astro-site/src/content/news/ \
  | sed 's/wpId: //' | sort -n > /tmp/content_wpids.txt

# Extract all p= values from netlify.toml
grep '{ p = "' /Users/chpapa/Workspaces/isoc.hk/astro-site/netlify.toml \
  | grep -o '"[0-9]*"' | tr -d '"' | sort -n > /tmp/netlify_pids.txt

# IDs in content but NOT in netlify.toml redirects
comm -23 /tmp/content_wpids.txt /tmp/netlify_pids.txt > /tmp/missing_pids.txt
wc -l /tmp/missing_pids.txt
cat /tmp/missing_pids.txt
```

For each missing `wpId`, find the post slug:
```bash
cat /tmp/missing_pids.txt | while read id; do
  grep -rl "^wpId: $id$" /Users/chpapa/Workspaces/isoc.hk/astro-site/src/content/news/ \
    | xargs basename 2>/dev/null
done
```

### Step 5 — Check pages collection coverage

The live WP site had static "pages" (not posts) like `/about/`, `/contact/`, `/disclaimer/`, etc. These live in `src/content/pages/`. Verify they all have corresponding `dist/` directories:

```bash
ls /Users/chpapa/Workspaces/isoc.hk/astro-site/src/content/pages/ \
  | sed 's/\.md$/\//' > /tmp/page_slugs.txt

cat /tmp/page_slugs.txt | while read slug; do
  if [ ! -d "/Users/chpapa/Workspaces/isoc.hk/astro-site/dist/$slug" ]; then
    echo "MISSING in dist: /$slug"
  fi
done
```

### Step 6 — Classify missing URLs

For each URL found in Step 3, classify it:

- **Missing content** — the WP post/page was not migrated to Astro (check `src/content/`)
- **Missing redirect** — the content exists but there's no redirect from the WP URL pattern
- **Intentional drop** — e.g., `/feed/`, `/wp-sitemap.xml`, search result pages (these can be OK to 404/redirect)
- **Category/tag archive** — WP archives like `/tag/ipv6/` that Astro may not implement

Note: `/feed/`, `/comments/feed/`, `/wp-sitemap.xml` are expected to be handled by a catch-all or to return 404 intentionally.

## If you get stuck

- If the live WP sitemap returns 404: the site may use a plugin like Yoast with a non-standard URL. Try `https://www.isoc.hk/wp-sitemap.xml` and `https://www.isoc.hk/sitemap_index.xml`.
- If the sitemap is very large (hundreds of entries): fetch it in sections using `<loc>` tag parsing.
- The `comm` tool requires both inputs to be sorted identically. If paths have inconsistent trailing slashes, normalize first with `sed 's|/*$|/|'`.
- WP archives (`/author/`, `/tag/`, `/20XX/`) are intentionally covered by pattern-based redirects in `netlify.toml` — treat these as covered and exclude from the diff.
